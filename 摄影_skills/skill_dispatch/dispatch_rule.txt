一、场景识别规则
================================================================================

场景识别是整个诊断流程的第一步，由 main_agent 调用 CLIP 模型进行自动分类。
识别结果决定了后续由哪个专业Agent接管诊断任务。

1.1 五大场景分类体系
--------------------------------------------------------------------------------

  场景ID        场景名称        对应Agent            分类优先级
  ---------     ----------      ---------------      ----------
  portrait      人像摄影        portrait_agent       P1
  landscape     风光摄影        landscape_agent      P1
  documentary   纪实摄影        documentary_agent    P1
  starry        星空摄影        starry_agent         P1
  still_life     静物摄影        still_life_agent     P1

1.2 场景判断逻辑（决策树）
--------------------------------------------------------------------------------

规则优先级：CLIP置信度 > 关键词匹配 > 默认兜底

第一步：CLIP模型推理
  - 将上传图像送入CLIP ViT-L/14@336px模型
  - 使用5个场景的预设标签向量进行余弦相似度计算
  - 取最高相似度作为CLIP分类结果
  - 阈值：scene_confidence >= 0.75 → 确认分类
  - 阈值：0.50 <= scene_confidence < 0.75 → 进入关键词辅助判断
  - 阈值：scene_confidence < 0.50 → 进入多模型投票

第二步：关键词辅助判断（CLIP置信度不足时）
  - 提取图像CLIP特征 + 用户输入文本关键词
  - 综合打分：score = 0.6 * clip_sim + 0.4 * keyword_match
  - 关键词匹配规则见下表

第三步：多模型投票（综合置信度仍不足时）
  - 使用GPT-4o Vision进行补充分类
  - 取CLIP结果与GPT-4o Vision结果的一致分类
  - 若两者不一致，取置信度更高的结果

第四步：兜底策略
  - 若所有方法均无法确定场景，默认使用 main_agent 进行通用诊断

1.3 各场景判断关键词表
--------------------------------------------------------------------------------

【人像摄影 - portrait】
  主要特征：
    - 画面中存在清晰可见的人脸/人物
    - 人物占据画面显著比例（面部面积 >= 画面5%）
    - 人物为画面主体而非背景元素

  CLIP正向标签：
    "portrait photo", "person close-up", "face portrait", "studio portrait",
    "outdoor portrait", "couple photo", "group portrait", "wedding photo",
    "fashion photography"

  CLIP负向标签：
    "landscape without people", "empty room", "food on table", "night sky",
    "product photo", "macro photography"

  文本关键词（权重0.4）：
    P1（强匹配）：人像、肖像、写真、证件照、婚纱照、情侣照、全家福、脸、面部、表情
    P2（中匹配）：半身照、全身照、特写、大头照、合照、闺蜜照、毕业照、cosplay
    P3（弱匹配）：人、模特、摄影师、拍摄对象

  排除规则：
    - 若人物面积 < 画面2% 且不在画面视觉重心区域 → 降级为其他场景
    - 若人物为背景装饰且画面主体为建筑/风景 → 分类为landscape
    - 若人物在运动场景中且以运动瞬间为主 → 分类为documentary

  特殊子场景自动标记：
    - "证件照"：面部正面、纯色背景、面部占比 > 30%
    - "写真照"：多角度、多姿态、有道具和场景设计
    - "环境肖像"：人物与环境有叙事关联，环境信息丰富

--------------------------------------------------------------------------------

【风光摄影 - landscape】
  主要特征：
    - 画面以自然/城市景观为主体
    - 天际线、地平线明显或可识别
    - 前景/中景/远景存在明显的层次关系
    - 宽画幅（16:9或更宽）优先权重

  CLIP正向标签：
    "landscape photography", "mountain scenery", "ocean sunset", "city skyline",
    "aerial photography", "waterfall photo", "forest scene", "desert landscape",
    "aurora borealis", "rainbow over field"

  CLIP负向标签：
    "close-up face", "studio product shot", "indoor still life", "food plate",
    "macro insect", "portrait background"

  文本关键词（权重0.4）：
    P1（强匹配）：风景、风光、山川、大海、天空、云海、日出、日落、城市风光、建筑
    P2（中匹配）：全景、航拍、黄昏、清晨、海边、湖面、田野、草原、雪山、瀑布
    P3（弱匹配）：景色、美景、旅游照、外景、自然

  排除规则：
    - 若画面以星空/银河为主体（天空占比>70%且可见星点）→ 重新分类为starry
    - 若画面中有大面积文字/广告牌/商品展示 → 重新分类为still_life（产品）
    - 若画面中的建筑被作为人像背景 → 分类为portrait

  特殊子场景自动标记：
    - "城市风光"：建筑占比 > 40%，可见天际线
    - "日出日落"：色温极度偏暖（>7000K）且太阳可见
    - "航拍风光"：俯视角度 > 60度

--------------------------------------------------------------------------------

【纪实摄影 - documentary】
  主要特征：
    - 画面捕捉的是"正在发生的事件"或"人物的真实生活状态"
    - 存在叙事性/故事性要素
    - 画面中人物活动是核心内容（非摆拍/非影棚）
    - 即时性、现场感、偶然性

  CLIP正向标签：
    "street photography", "documentary photo", "candid photo", "news photo",
    "travel photography", "market scene", "festival crowd", "sports action",
    "protest demonstration", "daily life"

  CLIP负向标签：
    "studio portrait", "posed photo", "arranged still life", "food styling",
    "product advertising", "fashion model standing"

  文本关键词（权重0.4）：
    P1（强匹配）：纪实、街拍、新闻、人文、旅行、节庆、民俗、运动、纪实摄影
    P2（中匹配）：街头、市场、集市、人群、游行、比赛、演唱会、婚礼纪实
    P3（弱匹配）：生活、日常、故事、瞬间、现场

  排除规则：
    - 若画面为明显的影棚摆拍 → 重新分类为portrait或still_life
    - 若画面为摆拍的美食/产品 → 重新分类为still_life
    - 若画面为摆拍的人像写真 → 重新分类为portrait

  特殊子场景自动标记：
    - "街拍"：户外、非正式场合、人物 unaware of camera
    - "运动纪实"：动态模糊 > 阈值 或 快门速度 < 1/500s
    - "旅行纪实"：多文化元素、标志性景点、异域风情

--------------------------------------------------------------------------------

【星空摄影 - starry】
  主要特征：
    - 画面主体为天空中的星体（星星/银河/星轨/极光/月亮）
    - 可见明显的星点或星轨轨迹
    - 通常在夜间或极暗光环境下拍摄
    - 可能包含前景元素（地景/人物/建筑剪影）

  CLIP正向标签：
    "milky way photo", "star trail photography", "northern lights aurora",
    "astrophotography", "night sky stars", "meteor shower", "lunar eclipse",
    "solar eclipse", "star field deep sky", "constellation photo"

  CLIP负向标签：
    "daytime landscape", "city lights without stars", "portrait outdoor",
    "indoor scene", "product photography"

  文本关键词（权重0.4）：
    P1（强匹配）：星空、银河、星轨、极光、流星、天文摄影、星空摄影
    P2（中匹配）：夜空、星星、月亮、月蚀、日蚀、星云、深空、星座
    P3（弱匹配）：夜景、夜晚、暗光、长时间曝光

  排除规则：
    - 若画面为城市夜景但不可见星点 → 重新分类为landscape（城市夜景）
    - 若画面为夜景人像（以人物为主体）→ 重新分类为portrait
    - 若画面中仅偶然包含少量星点但主体是地面风景 → 分类为landscape

  特殊子场景自动标记：
    - "银河拱桥"：可见银河弧形结构，跨度 > 90度
    - "星轨"：星点呈现明显弧线轨迹
    - "极光"：绿色/紫色光带可见，高度角 > 15度
    - "深空天文"：不可见地景，纯天体摄影

--------------------------------------------------------------------------------

【静物摄影 - still_life】
  主要特征：
    - 画面主体为无生命物体（食物/花卉/产品/珠宝/手表等）
    - 主体通常在受控环境下精心摆放
    - 注重质感、色彩、光影的精确呈现
    - 可能使用微距镜头拍摄细节

  CLIP正向标签：
    "still life photography", "food photography", "product photography",
    "macro flower", "jewelry close-up", "watch product shot", "table setting",
    "flat lay photo", "arranged objects", "commercial product"

  CLIP负向标签：
    "outdoor landscape", "street scene", "portrait face", "night sky",
    "candid photo", "action sports"

  文本关键词（权重0.4）：
    P1（强匹配）：静物、微距、美食摄影、产品摄影、花卉摄影、珠宝摄影、手表摄影
    P2（中匹配）：食物摆盘、桌面、茶几、书桌、橱窗、商品照、广告片、菜谱配图
    P3（弱匹配）：特写、细节、质感、纹理、摆拍、道具

  排除规则：
    - 若画面为户外自然生长的花卉/植物且无人工布置痕迹 → 分类为landscape（自然微距）
    - 若画面为户外人物手持物品的特写 → 分类为portrait（细节特写）
    - 若画面为商场橱窗展示但距离过远 → 可能分类为documentary

  特殊子场景自动标记：
    - "美食摄影"：可见食物/餐具/烹饪过程
    - "产品摄影"：商品主体明确，背景干净
    - "微距摄影"：放大倍率 > 1:1，极浅景深
    - "花卉静物"：花朵为主体，有布置痕迹


二、Agent分配矩阵
================================================================================

  场景ID        分配Agent              并行允许    超时(秒)    失败兜底
  ---------     ---------------        --------    --------    --------
  portrait      portrait_agent         是          90          main_agent
  landscape     landscape_agent        是          90          main_agent
  documentary   documentary_agent      是          90          main_agent
  starry        starry_agent           是          90          main_agent
  still_life    still_life_agent       是          90          main_agent
  unknown       main_agent             否          120         系统报错

跨场景组合处理规则：
  - 单张照片仅分类到单一场景（最高置信度场景）
  - 多张照片批量上传时，每张照片独立分类、可并行诊断
  - 若照片内容跨越多个场景（如"人像+风光"环境肖像）：
    优先级规则：portrait > landscape > documentary > starry > still_life
    辅助Agent：主场景Agent完成后，可选调用辅助场景Agent补充分析


三、任务流转协议
================================================================================

3.1 标准流程（5步串行+并行混合）

  ┌─────────────────────────────────────────────────────────────────┐
  │                    任务流转协议 v1.0                            │
  ├─────────────────────────────────────────────────────────────────┤
  │                                                                 │
  │  Step 1: 场景识别 (main_agent)                                 │
  │    ├─ 输入：原始图像 + 用户请求文本                              │
  │    ├─ 处理：CLIP推理 → 关键词匹配 → 场景确认                   │
  │    ├─ 输出：{scene_id, confidence, sub_scene_tags}             │
  │    ├─ 超时：15秒                                                │
  │    └─ 失败处理：使用GPT-4o Vision兜底分类                       │
  │                    │                                            │
  │                    ▼                                            │
  │  ┌─────── Step 2 ──────────┬─────── Step 3 ──────────┐        │
  │  │ 构图诊断 (scene_agent)   │ 风格对标 (style_agent)   │        │
  │  │ ├─ OpenCV量化分析        │ ├─ CLIP大师作品匹配      │        │
  │  │ ├─ LLM语义诊断          │ ├─ 5维度风格对比         │        │
  │  │ ├─ 构图/光影/色彩评估    │ ├─ 差距分析              │        │
  │  │ └─ 清晰度评估           │ └─ 改进路径              │        │
  │  │ [可并行] 90秒            │ [可并行] 60秒            │        │
  │  └────────────┬─────────────┴────────────┬────────────┘        │
  │               │                          │                      │
  │               └──────────┬───────────────┘                      │
  │                          ▼                                      │
  │  Step 4: 修图建议 (retouch_agent)                               │
  │    ├─ 输入：Step2诊断结果 + Step3对标结果                        │
  │    ├─ 处理：生成场景化修图方案                                   │
  │    ├─ 输出：全局调整参数 + 局部精修方案 + 预设推荐 + 工作流      │
  │    ├─ 超时：90秒                                                │
  │    └─ 失败处理：仅输出基础调整参数（跳过精细方案）               │
  │                    │                                            │
  │                    ▼                                            │
  │  Step 5: PDF报告生成 (pdf_export_agent)                         │
  │    ├─ 输入：全部诊断结果 + 原始图像                              │
  │    ├─ 处理：模板渲染 → 图像标注 → 排版 → PDF生成               │
  │    ├─ 输出：PDF/A-3 格式报告文件                                │
  │    ├─ 超时：60秒                                                │
  │    └─ 失败处理：输出Markdown格式纯文本报告（降级方案）           │
  │                                                                 │
  └─────────────────────────────────────────────────────────────────┘

3.2 任务状态定义

  状态名称        状态码    说明
  ----------      ------    ----
  PENDING         P         等待执行
  RUNNING         R         正在执行
  COMPLETED       C         执行完成
  FAILED          F         执行失败（可重试）
  FATAL           X         致命错误（不可重试）
  SKIPPED         S         因前置条件不满足被跳过
  CANCELLED       D         被用户取消

3.3 任务依赖关系

  Step 1 → Step 2（必须完成）
  Step 1 → Step 3（必须完成）
  Step 2 + Step 3 → Step 4（两者均须完成）
  Step 2 + Step 3 + Step 4 → Step 5（前三步均须完成）

3.4 快速模式路由

  用户可通过指定关键词触发快速模式，跳过部分步骤：

  请求模式                    执行步骤                    输出内容
  ----------------------      ----------                 ----------
  "只做构图分析"              Step1 → Step2(仅构图)      构图评分 + 构图建议
  "只看光影"                  Step1 → Step2(仅光影)      直方图 + 光影评分
  "帮我对标大师风格"          Step1 → Step3              风格雷达图 + 差距分析
  "给我修图参数"              Step1 → Step2 → Step4      诊断结果 + 修图参数
  "完整诊断"                  Step1→2→3→4→5             完整PDF报告
  "快速诊断"                  Step1→2(简化)→4(简化)     简化文字报告（无PDF）


四、异常处理
================================================================================

4.1 Agent级别异常

  异常类型                    处理策略
  ------------------------    ------------------------------------------------
  LLM API超时                 重试3次（间隔2秒/4秒/8秒指数退避），
                              超限后切换fallback模型（GPT-4o→通义千问）
  LLM API返回错误             解析错误类型，429限流→等待重试，
                              500服务错误→切换fallback
  CLIP推理失败                降级为纯关键词匹配分类
  SAM分割失败                跳过分割步骤，使用全图分析
  OpenCV分析异常              记录异常，使用LLM定性分析替代量化分析
  PDF生成失败                降级输出Markdown纯文本报告
  图像格式不支持              返回格式不支持提示，列出支持的格式列表
  图像文件损坏                返回文件损坏提示，建议重新上传
  图像尺寸过大                自动缩放至最大2048px后再分析（保留原EXIF）

4.2 流程级别异常

  异常类型                    处理策略
  ------------------------    ------------------------------------------------
  Step1场景分类失败           使用GPT-4o Vision兜底→仍失败则用main_agent通用诊断
  Step2诊断超时               返回已完成的部分分析结果 + 超时提示
  Step2+Step3均失败          返回Step1结果 + "诊断分析暂时不可用"提示
  Step4修图建议失败           返回Step2+3的诊断结论（无修图方案）
  Step5 PDF生成失败           降级输出Markdown纯文本报告

4.3 全局异常

  异常类型                    处理策略
  ------------------------    ------------------------------------------------
  系统内存不足                中止当前任务，释放缓存，返回"系统繁忙"提示
  GPU显存溢出                 切换到CPU推理模式（速度降低，功能不变）
  网络完全中断                仅返回CLIP+OpenCV本地分析结果（无LLM语义分析）
  API密钥失效                 返回配置错误提示，引导用户重新配置
  并发请求超限                返回队列提示，预计等待时间

4.4 错误重试机制

  重试级别        最大次数        间隔策略                触发条件
  ----------      ----------      ---------------        -----------------------
  L0 (即时)       1               无间隔                  偶发性网络抖动
  L1 (短期)       3               2s → 4s → 8s           API超时/500错误
  L2 (模型切换)   1               无间隔                  L1重试耗尽后切换模型
  L3 (降级)       0               N/A                     L2失败后进入降级模式

  降级模式优先级：
  1. 切换备用LLM（GPT-4o → 通义千问）
  2. 跳过非核心分析（SAM分割 → 全图分析）
  3. 简化输出格式（PDF → Markdown）
  4. 返回已完成的部分结果 + 错误说明
